## Theory

The 1's complement of a binary number flips every bit (`CPL A` does exactly this to the accumulator).
The 2's complement is the 1's complement plus 1 (`INC A`) — the standard way to represent a negative
number so that addition naturally handles subtraction. The program reads one input byte via `MOVX
A,@DPTR`, complements it, writes the 1's complement out, increments the accumulator for the 2's
complement, and writes that out too.

> **Note on this listing:** the manual's mnemonic table advances `DPTR` with `INC DPTR` **before** each
> `MOVX @DPTR,A` write, so tracing the program shows the 1's complement lands one byte **after** the
> input address, and the 2's complement one byte after that — e.g. input at `6500H` → 1's complement at
> `6501H` → 2's complement at `6502H`. (The manual's own printed output table labels these `6500H`/
> `6501H`, which would only be true if the writes happened *before* the `INC DPTR`s — a labelling slip
> against its own listing.) The addresses below are what the program you can actually run here produces.

## Registers / Instructions Used

| Mnemonic          | Meaning                                                                |
| ------------------ | ------------------------------------------------------------------- |
| `MOV DPTR,#6500H`  | Point the data pointer at the input byte.                             |
| `MOVX A,@DPTR`     | Read the input byte into the accumulator.                              |
| `CPL A`            | Complement every bit of the accumulator (1's complement).              |
| `INC DPTR`         | Advance the pointer to the next output slot.                           |
| `MOVX @DPTR,A`     | Write the 1's complement.                                              |
| `INC A`            | Add 1 to the accumulator (1's complement → 2's complement).            |
| `INC DPTR`         | Advance the pointer to the next output slot.                           |
| `MOVX @DPTR,A`     | Write the 2's complement.                                              |
| `RET`              | Return to the caller.                                                   |

## Sample Program

<CodeBlock
  lang="asm"
  code={`MOV DPTR, #6500H  ; point at the input byte
MOVX A, @DPTR      ; read it
CPL A              ; A = 1's complement
INC DPTR
MOVX @DPTR, A      ; write the 1's complement
INC A              ; A = 2's complement
INC DPTR
MOVX @DPTR, A      ; write the 2's complement
RET`}
/>

## Expected Output / Observation

| Address | Value | Meaning              |
| ------- | ----- | --------------------- |
| `6500H` | `03H` | Input (unchanged)      |
| `6501H` | `FCH` | 1's complement of 03H  |
| `6502H` | `FDH` | 2's complement of 03H  |
