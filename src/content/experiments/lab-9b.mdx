## Theory

The same step-and-delay subroutine (`1025h`) drives the motor for both directions — only the pattern
table it reads from differs. `DI` is pointed at the **forward table** (`09 05 06 0A`, at `103Fh`) for 20
steps, paused (`1037h`), then pointed at the **reverse table** (`0A 06 05 09`, at `1043h`, the same
bytes in reverse order) for another 20 steps. Feeding the pattern table backwards is what reverses the
rotor's physical direction — the stepping subroutine itself doesn't know or care which way it's
spinning.

## Registers / Instructions Used

| Mnemonic        | Meaning                                                                  |
| ---------------- | ---------------------------------------------------------------------------- |
| `MOV BL,20`      | Outer counter — number of steps to take in the current direction.            |
| `CALL 1025`      | Call the shared step-and-delay subroutine.                                   |
| `DEC BL` / `JNZ` | Loop the outer counter until 20 steps have been taken.                       |
| `CALL 103B`      | Call a longer pause subroutine between direction changes.                    |
| `MOV AL,[DI]` / `OUT C0,AL` | Inside the subroutine: fetch and write the next pattern (as in Lab 9A). |
| `RET`            | Return from a subroutine to its caller.                                      |

## Sample Program

<CodeBlock
  lang="asm"
  code={`; Address  Instruction
1000       MOV BL,20
1003       MOV DI,103F        ; forward table
1007       CALL 1025
100A       DEC BL
100C       JNZ 1003
100E       CALL 103B          ; long pause
1011       MOV BL,20
1014       MOV DI,1043        ; reverse table
1018       CALL 1025
101B       DEC BL
101D       JNZ 1014
101F       CALL 103B
1022       JMP 1000

1025       MOV CL,04
1028       MOV AL,[DI]
102A       OUT C0,AL
102C       MOV DX,1010
1030       DEC DX
1031       JNZ 1030
1033       INC DI
1034       LOOP 1028
1036       RET

1037       MOV DX,0FFFF       ; long inter-direction pause
103B       DEC DX
103C       JNZ 103B
103E       RET

103F       FORWARD DATA  09 05 06 0A
1043       REVERSE DATA  0A 06 05 09`}
/>

## Expected Output / Observation

The rotor takes 20 steps clockwise (forward table), pauses noticeably longer than between individual
steps, then takes 20 steps counter-clockwise (reverse table), and repeats. Both `StepperSimControls`
blocks above drive the same visualiser — start Forward, stop it, then start Reverse to see the
direction readout flip from CW to CCW.
