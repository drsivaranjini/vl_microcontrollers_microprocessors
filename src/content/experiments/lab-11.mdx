## Theory

Factorial (`n! = n × (n−1) × ... × 1`) is computed iteratively: keep a running product in `R0`, a
countdown copy in `R1`, and on each pass multiply the running product by the (already-decremented)
countdown value, stopping once the countdown reaches 1.

## Registers / Instructions Used

| Mnemonic     | Meaning                                                                 |
| ------------ | ------------------------------------------------------------------------ |
| `MOV r0,#7`  | Seed the number whose factorial is being computed (7, here).             |
| `MOV r1,r0`  | Copy it into the countdown register.                                    |
| `SUBS r1,r1,#1` | Decrement the countdown, updating flags.                              |
| `CMP r1,#1`  | Check whether the countdown has reached 1 (loop's stop condition).       |
| `BEQ STOP`   | Branch to `STOP` once the countdown reaches 1.                          |
| `MUL r3,r0,r1` | Multiply the running product by the new countdown value.               |
| `MOV r0,r3`  | Store the product back into `r0` for the next iteration.                |
| `BNE FACT`   | Otherwise, branch back to repeat.                                        |

## Sample Program

<CodeBlock
  lang="armasm"
  code={`AREA FACTORIAL, CODE, READONLY
EXPORT start
start
  MOV r0, #7        ; STORE FACTORIAL NUMBER IN R0
  MOV r1, r0        ; MOVE THE SAME NUMBER IN R1
FACT
  SUBS r1, r1, #1   ; SUBTRACTION
  CMP r1, #1        ; COMPARISON
  BEQ STOP
  MUL r3, r0, r1    ; MULTIPLICATION
  MOV r0, r3        ; Result
  BNE FACT          ; BRANCH TO THE LOOP IF NOT EQUAL
STOP
  NOP
  NOP
  NOP
  END               ; Mark end of file`}
/>

## Expected Output / Observation

With `r0` seeded to 7, the loop computes `7 × 6 × 5 × 4 × 3 × 2 = 5040`, left in `R0` when the program
reaches `STOP`.
