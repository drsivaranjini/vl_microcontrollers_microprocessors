## Theory

The first two terms (`0`, `1`) are written directly to internal RAM starting at `20H`, addressed
indirectly through `R1` (`@R1`). Each loop iteration reads the *previous* term (`R1` stepped back by one)
into the accumulator, adds the term before that, and stores the new term two slots ahead of where it
started â€” walking `R1` forward by the same two-step pattern each pass. `R0` counts down the number of
additional terms to generate, and `DJNZ R0,BACK` repeats until it reaches zero.

## Registers / Instructions Used

| Mnemonic          | Meaning                                                                 |
| ------------------ | ----------------------------------------------------------------------- |
| `MOV R0,#0AH`      | Number of additional terms to generate (10, here).                       |
| `MOV R1,#20H`       | Point `R1` at the start of the series in internal RAM.                    |
| `MOV @R1,#00H`      | Seed the first term (`0`) at `20H`.                                       |
| `INC R1`            | Advance to the next slot.                                                 |
| `MOV @R1,#01H`       | Seed the second term (`1`) at `21H`.                                      |
| `MOV A,@R1`         | Read the most recent term (indirect via `R1`).                            |
| `DEC R1`            | Step back to the term before that.                                        |
| `ADD A,@R1`         | Add it to the accumulator â€” `A` now holds the new term.                    |
| `INC R1` (Ă—2)       | Step forward two slots (past the "before" term and the "recent" term).    |
| `MOV @R1,A`         | Store the new term.                                                       |
| `DJNZ R0,BACK`       | Decrement `R0`; loop while more terms remain.                             |
| `SJMP HERE`          | Infinite self-loop â€” halt.                                                |

## Sample Program

<CodeBlock
  lang="asm"
  code={`MOV R0,#0AH    ; 10 more terms to generate
MOV R1,#20H    ; series starts at internal RAM 20H
MOV @R1,#00H   ; term[0] = 0
INC R1
MOV @R1,#01H   ; term[1] = 1
BACK:
MOV A,@R1      ; A = most recent term
DEC R1
ADD A,@R1      ; A += term before that
INC R1
INC R1
MOV @R1,A      ; store the new term
DJNZ R0,BACK   ; loop for the remaining terms
HERE:
SJMP HERE      ; halt`}
/>

> **Note:** the manual's own opcode column prints `7A` for `MOV R1,#20H` â€” that is actually the encoding
> for `MOV R2,#20H` (register R2, not R1). Every other instruction in this same listing (`INC R1`, `MOV
> A,@R1`, `DEC R1`, `MOV @R1,A`) is unambiguously R1-based and matches the manual's own prose exactly, so
> this looks like an isolated transcription slip in the manual's opcode table. The pre-assembled program
> above uses the correct `MOV R1,#20H` encoding (`79 20`) â€” verified by running it and confirming it
> produces the Fibonacci sequence below.
