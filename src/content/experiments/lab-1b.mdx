## Theory

Eight bytes starting at `1400H` are copied, one at a time, to a destination block starting at `1450H`.
`CL` holds the count (8), `SI` walks the source block, `DI` walks the destination block. Each pass loads
a byte via `LODSW` into `AL` (auto-incrementing `SI`), stores it at `[DI]`, advances `DI`, decrements the
counter, and loops via `JNZ` while the counter is still nonzero.

## Registers / Instructions Used

| Mnemonic       | Meaning                                                          |
| -------------- | ------------------------------------------------------------------ |
| `MOV CL,08H`   | Set the loop counter to 8 (number of bytes to copy).                |
| `MOV SI,1400H` | Point the source index at the start of the source block.            |
| `MOV DI,1450H` | Point the destination index at the start of the destination block.  |
| `LODSW`        | Load from `[SI]` into `AX` (`AL` holds the byte here); auto-increment `SI`. |
| `MOV [DI],AL`  | Store the low byte to the destination address.                      |
| `INC DI`       | Advance the destination pointer by one byte.                        |
| `DEC CL`       | Decrement the remaining-byte counter.                                |
| `JNZ Back`     | Loop back while the counter hasn't reached zero.                    |
| `HLT`          | Halt — end of program.                                                |

## Sample Program

<CodeBlock
  lang="asm"
  code={`MOV CL,08H     ; 8 bytes to copy
MOV SI,1400H   ; source pointer
MOV DI,1450H   ; destination pointer
Back:
LODSW          ; load byte at [SI] into AL, auto-increment SI
MOV [DI],AL    ; store AL to destination
INC DI         ; advance destination pointer
DEC CL         ; one fewer byte remaining
JNZ Back       ; loop while CL != 0
HLT            ; end of program`}
/>

## Expected Output / Observation

After running, the 8 bytes at `1450H`–`1457H` should be an exact copy of the 8 bytes originally at
`1400H`–`1407H`.
