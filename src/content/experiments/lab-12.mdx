## Theory

The array (7 words, starting at `0x40000000`) is scanned once: `R3` holds the running extremum,
starting as the first element. Each subsequent element is loaded into `R4` and compared against `R3`;
the branch condition (`BLS` — branch if lower-or-same — in the listing below) decides whether `R3` is
replaced. Using **`BLS`** keeps the *smaller* of the two in `R3` at the end (branch away, i.e. keep the
current R3, when R3 is already the larger one is *not* what BLS does — see the viva question below for
exactly how the comparison direction determines smallest vs. largest); swapping it for **`BHI`** (branch
if higher) makes the same routine find the *largest* element instead.

## Registers / Instructions Used

| Mnemonic        | Meaning                                                                   |
| --------------- | ---------------------------------------------------------------------------- |
| `MOV R5,#06`     | Loop counter — 6 more comparisons after the first element is loaded (7 total). |
| `MOV R1,#0x40000000` | Base address of the array in memory.                                     |
| `MOV R2,#0x4000001C` | Address where the result will be stored.                                  |
| `LDR R3,[R1]`    | Load the first array element as the initial running extremum.               |
| `ADD R1,R1,#04`  | Advance the memory pointer to the next word (4 bytes).                       |
| `LDR R4,[R1]`    | Load the next element for comparison.                                        |
| `CMP R3,R4`      | Compare the running extremum against the new element.                        |
| `BLS LOOP1`      | Branch if R3 is lower-or-same — controls which value survives (see viva).    |
| `SUBS R5,R5,#01` | Decrement the loop counter.                                                  |
| `STR R3,[R2]`    | Store the final extremum to the result address.                              |

## Sample Program

### Part I — ARM assembly

<CodeBlock
  lang="armasm"
  code={`; ALP TO FIND THE LARGEST/SMALLEST NUMBER IN AN ARRAY OF 7 NUMBERS
AREA LAR_SMAL, CODE, READONLY
EXPORT start
start
  MOV R5, #06                ; COUNTER VALUE E.G 7 NUMBERS
  MOV R1, #0x40000000        ; START OF THE DATA MEMORY
  MOV R2, #0x4000001C        ; RESULT LOCATION
  LDR R3, [R1]                ; GET THE FIRST DATA
LOOP
  ADD R1, R1, #04            ; MEMORY POINTER UPDATED TO FETCH NEXT DATA
  LDR R4, [R1]                ; GET NEXT NUMBER
  CMP R3, R4                  ; COMPARE BOTH NUMBERS
  BLS LOOP1                   ; (use BHI instead of BLS to find the largest)
  MOV R3, R4
LOOP1
  SUBS R5, R5, #01            ; DECREMENT THE COUNTER
  CMP R5, #00
  BNE LOOP
  STR R3, [R2]
STOP
  B STOP
  END`}
/>

### Part II — C wrapper (calls the assembly routine)

<CodeBlock
  lang="c"
  code={`extern void start();
void main()
{
    start();
    while (1); // Stop
}`}
/>

## Expected Output / Observation

After running, memory address `0x4000001C` holds the smallest of the 7 values seeded into
`0x40000000..0x40000018`. Replacing `BLS` with `BHI` in the listing and re-running instead leaves the
**largest** value at the same result address.
