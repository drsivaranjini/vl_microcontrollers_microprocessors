## Theory

A 4-phase unipolar stepper motor rotates one step for each new 4-bit coil-energising pattern written to
its driver port. Cycling through the pattern table **09h → 05h → 06h → 0Ah** (repeating) rotates the
rotor clockwise, one step per pattern change. The 8086 writes each pattern to **port C0h** via `OUT`,
holds it for a software delay (`DEC DX` / `JNZ`), then advances to the next pattern — the delay count
sets the step rate (speed).

## Registers / Instructions Used

| Mnemonic       | Meaning                                                            |
| -------------- | --------------------------------------------------------------------- |
| `MOV DI,1100H` | Point `DI` at the pattern table in memory.                             |
| `MOV CL,04`    | Load a counter of 4 (one full cycle through the pattern table).       |
| `MOV AL,[DI]`  | Fetch the next pattern byte.                                          |
| `OUT C0,AL`    | Write the pattern to port C0h (the stepper driver).                    |
| `DEC DX` / `JNZ DELAY` | Software delay loop — its length sets the time each step is held. |
| `INC DI`       | Advance to the next entry in the pattern table.                        |
| `LOOP LOOP1`   | Decrement `CL`; repeat while nonzero (one pass through all 4 patterns). |
| `JMP 1000`     | Jump back to the start — run forever.                                  |

## Sample Program

<CodeBlock
  lang="asm"
  code={`; Address  Label   Instruction
1000       START   MOV DI, 1100
1004               MOV CL,04
1007       LOOP1   MOV AL,[DI]
1009               OUT C0,AL
100B               MOV DX,1010
100F       DELAY   DEC DX
1010               JNZ DELAY
1012               INC DI
1013               LOOP LOOP1
1015               JMP 1000
1018               TABLE 09 05 06 0A`}
/>

## Expected Output / Observation

The rotor advances one step clockwise each time a new pattern is written to C0h, cycling through
09-05-06-0A repeatedly. Increasing the `MOV DX,1010` delay value slows the rotation; decreasing it
speeds the rotation up. Use the **Speed** slider above to reproduce this — it controls the same
observable behaviour (step rate) that the delay loop controls in the real program.
