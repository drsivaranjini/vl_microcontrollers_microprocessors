## Theory

A sawtooth wave is generated by continuously writing an ever-increasing 8-bit value to a DAC. DAC0808
is memory-mapped on the 8051 board at **DAC1 = FFC0h** and **DAC2 = FFC8h**; `00h` corresponds to 0V and
`FFh` to +5V. Writing `00, 01, 02, ..., FF` to DAC2 in a tight loop, then wrapping back to `00`, produces
a linear ramp — a sawtooth — at the DAC's analogue output.

| Port  | MICRO-51 | MICRO-POWER 1 | MICRO-31EB |
| ----- | -------- | ------------- | ---------- |
| DAC 1 | FFC0h    | A0C0h         | FFC0h      |
| DAC 2 | FFC8h    | A0C8h         | FFC8h      |

## Registers / Instructions Used

| Mnemonic         | Meaning                                                     |
| ---------------- | ------------------------------------------------------------ |
| `ORG 4100`        | Set the program's starting address to 4100h.                |
| `MOV DPTR,#FFC8`  | Load the data pointer with DAC2's external-memory address.  |
| `MOV A,#00`       | Initialise the accumulator (the ramp value) to 0.            |
| `MOVX @DPTR,A`    | Write the accumulator to external memory at `[DPTR]` (DAC2). |
| `INC A`           | Increment the ramp value; wraps `FFh → 00h` automatically.  |
| `SJMP LOOP`       | Short jump back to repeat the ramp indefinitely.              |

## Sample Program

### To obtain 0V at DAC 1 (reference)

<CodeBlock
  lang="asm"
  code={`ORG 4100
MOV A,#3F        ; A = 3Fh (mid-scale reference)
MOV DPTR,#FFC0   ; point at DAC1
MOVX @DPTR,A     ; write once
HERE: SJMP HERE  ; halt`}
/>

### Sawtooth waveform at DAC 2

<CodeBlock
  lang="asm"
  code={`ORG 4100
MOV DPTR,#FFC8   ; point at DAC2
MOV A,#00        ; start ramp at 0
LOOP: MOVX @DPTR,A   ; write ramp value to DAC2
INC A            ; A = A + 1 (wraps FFh -> 00h)
SJMP LOOP        ; repeat forever`}
/>

## Expected Output / Observation

DAC2's output rises linearly from 0V to just under +5V, then drops sharply back to 0V and repeats — a
sawtooth waveform. The oscilloscope widget above reproduces exactly this trace from the real `MOVX`
writes the emulator executes.
